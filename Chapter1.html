Chapter 1: Getting Started with JavaScript - Summary

1. Introduction to JavaScript

  • JavaScript works on all major web browsers and allows real-time code execution.
  • It can be used not just for web pages, but also for full-stack, mobile, and desktop applications.
  • Supports various open APIs, libraries, and frameworks for enhanced functionality.
  • Developers can modify or extend existing code to build dynamic web applications.

2. JavaScript vs. Java

  • Despite the similar names, JavaScript and Java are completely different languages.
  • Originally called LiveScript, it was created in just 10 days by Netscape to make web pages interactive.
  • Renamed to JavaScript by Sun Microsystems to leverage Java’s popularity at the time.

3. ECMAScript Standardization

  • Microsoft introduced JScript, leading to compatibility issues between browsers.
  • ECMA International standardized JavaScript as ECMAScript to unify scripting syntax.
  • JavaScript is the most widely used implementation of the ECMAScript specification.
  • ECMAScript receives annual updates with modern features and syntax improvements.

4. Why ES6 Matters

  • Early JavaScript was simple and mainly used for small browser interactions.
  • With HTML5, JavaScript evolved into a full-scale development language.
  • ES6 (ECMAScript 2015) introduced modules, classes, arrow functions, and block scoping.
  • Each new ECMAScript version continues refining and expanding JavaScript’s capabilities.

5. JavaScript in Web Development

  • Websites operate through client-server communication — clients send requests, servers respond.
  • Front-end development covers everything users see and interact with.
  • JavaScript powers front-end logic, animations, form validation, and dynamic data exchange.
  • Frameworks like React, Angular, and Vue are built on JavaScript foundations.

6. Using the Browser Console

  • Every web browser includes a built-in JavaScript engine.
  • Open the console with Ctrl + Shift + J (Windows) or Option + Command + J (macOS).
  • Examples:
    • 10 + 20
    • console.log("Hello, World!")

7. Setting Up a Development Environment

  • Install Visual Studio Code (VS Code) for coding in JavaScript and other languages.
  • Download and install the D2Coding font for Korean and English support.
  • Recommended VS Code Extensions:
    • Live Server — auto-refresh web pages upon saving code.
    • Prettier — automatically formats code neatly.
    • Material Theme and Material Icon Theme — improve interface visuals.

8. Writing JavaScript Code

  • JavaScript can be written in three ways:
    • Inline script — directly in an HTML tag:
      <button onclick="alert('Message!')">Click!</button>
    • Internal script — inside <script> tags within HTML:
      <script>
      function random(n) { return Math.floor(Math.random() * n); }
      document.body.style.backgroundColor = 'rgb(' + random(255) + ',' + random(255) + ',' + random(255) + ')';
      </script>
    • External script — linked via a separate .js file:
      <script src="js/script.js"></script>

9. Advantages of External Scripts

  • Allows code reuse across multiple web pages.
  • Improves project organization and maintainability.
  • Separates JavaScript from HTML structure for cleaner code.

10. How Browsers Process JavaScript

  • 1️⃣ The browser reads and interprets HTML structure.
  • 2️⃣ It analyzes tag order and nesting relationships.
  • 3️⃣ External stylesheets are loaded and applied.
  • 4️⃣ Upon reaching a <script> tag, the JavaScript engine executes the code.
  • 5️⃣ The script dynamically updates or interacts with the web page.

11. Summary

  • JavaScript evolved from a small browser script into a powerful, standardized language.
  • It forms the foundation of modern front-end development.
  • Learning setup tools like VS Code and understanding script placement are key to starting effectively.
  • Knowing ES6+ syntax prepares developers for real-world web projects.
