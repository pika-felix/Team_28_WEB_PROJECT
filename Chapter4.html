Chapter 4: Functions and Scope - Summary

1. Importance of Functions

  • Functions are the “core of programming,” allowing complex problems to be divided into smaller, manageable parts.
  • They promote reusability — instead of repeating code, developers can call the same function when needed.
  • Example: A To-Do List app can use separate functions to add, complete, and delete tasks.

2. Declaring and Calling Functions

  • Functions are defined using the function keyword:
    function greet() { console.log("Hello!"); }
  • To execute a function, use parentheses: greet();
  • Although functions can be placed anywhere in the code, they are typically written before being called.

3. Parameters, Arguments, and Return Values

  • Parameters allow functions to receive values from outside:
    function add(a, b) { return a + b; }
  • Arguments are the actual values passed when calling a function: add(3, 5);
  • The return keyword sends a result back to where the function was called.
  • ES6 introduced default parameters:
    function multiply(a, b = 5) { return a * b; }

4. Debugging with Developer Tools

  • Debugging is the process of finding and fixing errors in code.
  • Use browser developer tools (F12 → Sources tab) to set breakpoints and inspect variable values step by step.

5. Understanding Variable Scope

  • Scope determines where a variable can be accessed.
  • var → function scope; let and const → block scope.
  • Local variables are declared inside a function and cannot be accessed outside.
  • Global variables are declared outside functions and can be accessed anywhere.
  • Hoisting: JavaScript moves variable declarations to the top, which can lead to unexpected results.

6. Best Practices for Variables

  • Use let and const instead of var to avoid hoisting issues.
  • Minimize the use of global variables.
  • Use const for objects to prevent reassignment (properties can still change).

7. Function Expressions and IIFE

  • Functions can be stored in variables:
    let sum = function(a, b) { return a + b; };
  • IIFE (Immediately Invoked Function Expression) runs immediately after being defined:
    (function() { alert("Hello!"); })();

8. Arrow Functions (ES6)

  • Provide a shorter syntax for function expressions:
    const hi = () => "Hello!";
  • If only one statement is inside, braces and return can be omitted.

9. Callback Functions

  • A function passed as an argument to another function.
  • Example:
    button.addEventListener("click", () => alert("Clicked!"));
  • Used in event handling and asynchronous programming.

10. Spread and Rest Syntax

  • Spread ( ... ) expands arrays or objects:
    console.log(...[1, 2, 3]);
  • Rest ( ... ) collects multiple arguments:
    function addAll(...nums) { return nums.reduce((a,b)=>a+b); }

11. Timer Functions

  • Allow delayed or repeated execution of code.
  • setInterval() — runs repeatedly at intervals.
  • clearInterval() — stops a running interval.
  • setTimeout() — executes code after a specified time.
  • Example:
    setTimeout(() => console.log("Hello after 3s"), 3000);

12. Summary

  • Functions improve code structure, reusability, and readability.
  • Understanding scope helps prevent variable conflicts and errors.
  • Modern JavaScript features (arrow functions, spread/rest, etc.) make code more concise.
  • Timers and callbacks enable interactive, dynamic web applications.
