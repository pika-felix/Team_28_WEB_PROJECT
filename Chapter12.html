Chapter 12: HTTP Communication and JSON - Summary

1. What is HTTP and HTTPS?

  • HTTP is the protocol used for communication between clients and servers.
  • HTTPS is the secure version of HTTP that encrypts data.
  • Clients send HTTP requests, and servers send HTTP responses with data.

2. Request and Response Headers

  • Request headers: contain browser info, OS, language, etc.
  • Response headers: include content type, response time, and other metadata.
  • The real webpage content appears in the response body.

3. GET vs POST Methods

  • GET: sends data in the URL, not secure, no request body.
  • POST: sends data in the request body, used for login forms and sensitive data.

4. HTTP Response Status Codes

  • 200 OK — Success
  • 400 Bad Request — Invalid request
  • 401 Unauthorized — Login required
  • 403 Forbidden — Access denied
  • 404 Not Found — File not found
  • 500 Internal Server Error — Server failure

5. JSON (JavaScript Object Notation)

  • Lightweight text format widely used for data transfer.
  • Uses key–value pairs with double quotes around keys.
  • Does not allow functions.
  • Supports numbers, strings, arrays, objects, booleans, and null.

6. Converting JSON and JavaScript Objects

  • JSON.stringify() — converts an object to a JSON string.
  • JSON.parse() — converts a JSON string back to an object.

7. Asynchronous HTTP Requests (AJAX)

  • Allows updating parts of a webpage without reloading it.
  • Commonly used for dynamic websites like Facebook and Twitter.

8. XMLHttpRequest (Old Method)

  • Uses readyState, status, and responseText to handle responses.
  • Requires manual setup and callbacks.

9. Fetch API (Modern Method)

  • Simpler and more readable than XHR.
  • Returns Promises for async operations.
  • Common usage:
    fetch(url).then(res => res.json())
  • Supports async/await for cleaner code.

10. Displaying JSON Data in Web Pages

  • JSON data can be looped through using forEach() or other array methods.
  • Data can be inserted into HTML dynamically.

11. Error Handling (try, catch, finally)

  • Prevents the program from crashing on errors.
  • try — code to attempt.
  • catch — handles errors.
  • finally — runs regardless of success or failure.
  • Supports custom errors with throw new Error().

12. Summary

  • HTTP enables communication between clients and servers.
  • GET and POST support different types of data transmission.
  • JSON is the universal format for sending and receiving data.
  • Fetch API is the modern way to perform async web requests.
  • Error handling ensures stability in real-world web applications.
