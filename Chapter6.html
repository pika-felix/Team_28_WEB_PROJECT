Chapter 6: Events and Event Handling - Summary

1. What is an Event?

  • An event is any action that occurs in a web browser, triggered by users or the system.
  • Examples include clicking, key presses, mouse movement, form submissions, and page loads.
  • Events allow web pages to respond dynamically to user interactions.

2. Types of Events

  • Document events: When a web page is loaded or closed.
  • Mouse events: Clicking, hovering, moving, or scrolling the mouse.
  • Keyboard events: Pressing or releasing a key.
  • Form events: When users input or submit data in a form.

3. Event Handlers (Ways to Handle Events)

  • Inline HTML event:
    <button onclick="alert('Clicked!')">Click</button>
    Simple but not recommended for larger projects.
  • Assigning events in JavaScript:
    button.onclick = function() { document.body.style.backgroundColor = 'green'; }
  • Using addEventListener(): (modern, preferred method)
    button.addEventListener("click", () => { document.body.style.backgroundColor = "green"; });

4. Example: Counting Text Characters

  • Counts characters entered in a text field when a button is clicked.
  • Example:
    const button = document.querySelector("#bttn");
    button.addEventListener("click", () => {
      const word = document.querySelector("#word").value;
      document.querySelector("#result").innerText = word.length;
    });

5. Example: Creating a Modal Box

  • A modal box is a pop-up window that focuses the user’s attention.
  • Displayed or hidden by adding or removing an active class.
  • Example:
    open.addEventListener("click", () => modalBox.classList.add("active"));
    close.addEventListener("click", () => modalBox.classList.remove("active"));

6. The event Object

  • Contains information about the event that occurred.
  • Common properties:
    • event.target — the element that triggered the event
    • event.pageX, event.pageY — mouse coordinates
    • event.key, event.code — pressed key information
  • Example:
    box.addEventListener("click", (e) => alert(`Clicked at: ${e.pageX}, ${e.pageY}`));

7. Example: Image Carousel

  • A simple slideshow that changes images using arrow buttons.
  • Example:
    const pics = ["pic-1.jpg","pic-2.jpg","pic-3.jpg"];
    let i = 0;
    arrows.forEach(arrow => {
      arrow.addEventListener("click", (e) => {
        if (e.target.id === "left") i--;
        else i++;
        if (i < 0) i = pics.length - 1;
        if (i >= pics.length) i = 0;
        container.style.backgroundImage = `url(images/${pics[i]})`;
      });
    });

8. Event Propagation

  • Event Bubbling: The event moves upward from the target to its parent elements (default behavior).
  • Event Capturing: The event travels downward from the root to the target (enabled with useCapture = true).
  • Example:
    element.addEventListener("click", e => console.log(e.target.tagName), true);

9. event.target vs. event.currentTarget

  • event.target: The element that originally triggered the event.
  • event.currentTarget: The element currently processing the event during bubbling or capturing.

10. Summary

  • Events connect user actions to JavaScript behavior, enabling interactive web pages.
  • addEventListener() is the best practice for handling events.
  • The event object provides detailed data about user interactions.
  • Advanced examples include modal boxes, image carousels, and keyboard/mouse tracking.
  • Understanding event propagation helps control how events flow through nested elements.
