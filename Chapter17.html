Chapter 17: Using Web APIs - Summary

1. What Is an API?

  • An API (Application Programming Interface) allows different programs to communicate and share data.
  • APIs let developers use external features without building them from scratch.
  • Examples include Canvas API for graphics and Weather APIs for real-time data.

2. Public (Open) APIs

  • Public APIs are available for anyone to use.
  • They allow integration of services such as maps, authentication, and public data.
  • Many government and portal services provide open APIs.

3. Web APIs

  • Web APIs are built-in browser APIs that work with JavaScript.
  • They extend JavaScript’s capabilities without extra libraries.
  • MDN Web Docs provides documentation and examples.

4. Web Storage API

  • Web Storage stores data in the browser more securely than cookies.
  • Data is stored as JSON strings.
  • It is not automatically sent to the server.

5. Cookies vs Web Storage

  • Cookies: small, sent to servers automatically, limited size, privacy risks.
  • Web Storage: larger capacity, more secure, better performance.

6. Types of Web Storage

  • Session Storage: data exists while the browser tab is open.
  • Local Storage: data remains even after the browser is closed.

7. Storage Methods

  • setItem() – save data
  • getItem() – retrieve data
  • removeItem() – delete specific data
  • clear() – remove all data

8. Using Local Storage

  • Objects and arrays must be converted using JSON.stringify().
  • Stored data is restored using JSON.parse().
  • Common uses include saving user preferences and to-do lists.

9. To-Do List with Web Storage

  • Users can add, delete, and complete tasks.
  • Tasks remain saved after page refresh.
  • Uses DOM manipulation, events, and localStorage.

10. Geolocation API

  • Provides the user’s location (latitude and longitude).
  • User permission is required.
  • Works only in secure (HTTPS) environments.

11. Key Geolocation Methods

  • getCurrentPosition() – gets current location.
  • watchPosition() – tracks location changes.
  • clearWatch() – stops tracking.

12. Summary

  • Web APIs expand what JavaScript can do in the browser.
  • Web Storage enables persistent client-side data.
  • Geolocation supports location-based services with user consent.
  • Web APIs are essential for modern interactive web applications.
