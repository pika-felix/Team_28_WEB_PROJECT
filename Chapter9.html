Chapter 9: Creating JavaScript Objects - Summary

1. Understanding Objects

  • An object stores and processes related data and actions.
  • JavaScript objects group properties (data) and methods (functions).
  • There are two main types:
    • Built-in objects: Provided by JavaScript, like Math and Date.
    • User-defined objects: Created by programmers as needed.

2. Creating Objects

  • Create objects using {}:
    let book1 = { title: "Web Standard", pages: 648 };
  • Access properties with object.property or object["property"].
  • Add, modify, or delete properties:
    book1.author = "Koh Kyung-hee";
    delete book1.pages;

3. Nested Objects

  • Objects can contain other objects:
    let student = { name: "Doremi", score: { history: 85, science: 94 } };
  • Use this to refer to the current object in methods.

4. Object Methods

  • Functions defined inside objects are called methods.
  • Example:
    let book = { title: "JS Basics", buy() { console.log("Purchased"); } };
  • this refers to the current object:
    this.done ? "Finished" : "Reading";

5. Copying Objects

  • Primitive values copy by value.
  • Objects copy by reference, meaning both variables point to the same object.

6. Constructor Functions

  • Use constructor functions to create multiple objects:
    function Book(title, pages) { this.title = title; this.pages = pages; }
  • Create objects using new:
    let book1 = new Book("Web Standard", 648);

7. Using Classes (ES6)

  • ES6 introduced the class keyword for cleaner syntax:
    class Book { constructor(t, p) { this.title = t; this.pages = p; } }
  • Methods are added directly inside the class.

8. Accessing Keys and Values

  • Loop through object properties with:
    for (key in obj) { console.log(key, obj[key]); }
  • Or use:
    Object.keys(obj), Object.values(obj), Object.entries(obj)

9. Prototypes and Inheritance

  • Objects inherit features through prototypes.
  • Define shared methods using the constructorâ€™s prototype:
    Book.prototype.finish = function() { console.log("Done"); };
  • All objects created with the same constructor share this method.

10. Inheritance (Prototype & Class)

  • Prototype inheritance:
    Object.setPrototypeOf(Textbook.prototype, Book.prototype);
  • Class inheritance:
    class Textbook extends Book { constructor(t, p, m) { super(t, p); this.major = m; } }

11. Summary

  • Objects organize related data and behavior into one unit.
  • Methods define what objects can do.
  • Constructors and classes create object templates.
  • Prototypes and inheritance support code reuse and scalability.
